# Notes on Local LLM Serving Frameworks

The local LLM serving landscape offers distinct tradeoffs depending on your hardware and use case. For Apple Silicon users, MLX-LM delivers the best native performance since it is purpose-built for Apple's Metal GPU and unified memory architecture, though it is limited to macOS and supports fewer quantization formats (INT4, INT8, MXFP4) compared to the GGUF ecosystem. llama.cpp and Ollama both leverage the GGUF format, which has become the de facto standard for cross-platform quantized inference with an extensive range of quantization levels from Q2 through Q8 and beyond; Ollama adds a convenient CLI and OpenAI-compatible API on top of llama.cpp's Metal backend, while also previewing a native MLX backend for even better Apple Silicon performance. vLLM is the clear choice for high-throughput production serving on NVIDIA GPUs with its PagedAttention and continuous batching, and its new vllm-metal plugin brings Apple Silicon support, though it remains less mature than the CPU-focused alternatives. TGI from Hugging Face offers strong quantization options (GPTQ, AWQ, EXL2, FP8) and function calling, but lacks GPU acceleration on Apple Silicon, making it a CPU-only option on Macs. Ultimately, GGUF-based frameworks (llama.cpp, Ollama) provide the broadest compatibility and model availability, while MLX-LM and vLLM's Metal plugin offer the best performance for Apple Silicon users willing to accept narrower format support.
