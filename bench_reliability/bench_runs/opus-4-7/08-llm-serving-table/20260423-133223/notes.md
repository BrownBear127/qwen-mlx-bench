# Notes on Local LLM Serving Frameworks

The central tradeoff across these frameworks is between hardware reach, performance optimization, and ease of use. llama.cpp and its wrappers (Ollama, LocalAI, LM Studio) dominate the consumer space because GGUF quantization is well-supported across CPU, CUDA, and Apple Metal, giving broad hardware compatibility at the cost of not being as aggressively optimized for any single backend. vLLM targets high-throughput GPU serving with techniques like PagedAttention and continuous batching, making it the strongest choice for CUDA-equipped servers but leaving Apple Silicon and CPU-only users without support. MLX takes the opposite approach, being purpose-built for Apple Silicon with tight Metal integration and excellent performance on M-series chips, but it lacks an OpenAI-compatible server endpoint and tool-calling support out of the box. MLC LLM uses TVM-based compilation to generate optimized code for each target platform, achieving strong cross-platform coverage including mobile, though its quantization format ecosystem is less mature than GGUF. In general, users choosing a local framework must weigh serving features (tool calling, OpenAI API compatibility, batching) against raw inference speed and the specific hardware they intend to run on.
