Find a way to compute a meaningful "diff" between two SVG files (something more
useful than a raw text diff — e.g. structural diff, visual diff, attribute diff,
or rendered pixel diff). Write a minimal POC.

Setup:
  - Create two small SVG files: a.svg (a red circle) and b.svg (a blue circle of
    slightly different radius) — write them as part of the POC
  - Compute a diff and print a human-readable summary (e.g. "fill changed: red -> blue,
    radius changed: 50 -> 60")

Constraints:
  - If your POC needs third-party Python packages, write a requirements.txt file (one per line) so the verifier can install them
  - Output: poc.py + README.md
  - poc.py must be runnable with: uv run --no-project --with <lib> python poc.py
  - The diff output should highlight at least 2 changes between a.svg and b.svg
  - Pure-text-diff solutions (just running difflib on the file contents)
    are NOT acceptable; you need structural/visual awareness
  - README.md explains the approach you chose and why it's better than text diff
