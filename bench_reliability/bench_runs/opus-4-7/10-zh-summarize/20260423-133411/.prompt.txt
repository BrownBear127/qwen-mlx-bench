Write a minimal POC that summarizes a Traditional Chinese (繁體中文) article
to roughly 100 characters. The POC must run fully OFFLINE with no network calls
and use only Python stdlib (no transformers, no openai, no tiktoken, no llama-cpp,
nothing that requires network or large model downloads).

Approach hint: this is intentionally constrained — the goal is to test whether
you can produce something simple but reasonable (e.g. extractive summarization
via sentence ranking, or first-N-chars + last-N-chars heuristic, or TF-IDF on
character n-grams). It doesn't have to be brilliant; it just has to run cleanly
under the constraints and produce a 50-150 char output.

Setup:
  - Embed this 繁體中文 sample article in poc.py as a string variable (≥500 chars
    of real-looking traditional Chinese — e.g. discussion of 設計 / 文化 / 寫作 /
    工藝 / 語言 / 食物 anything coherent)
  - Run summarize() on it
  - Print the result and its character count

Output:
  - poc.py + README.md (README explains your summarization approach and why you
    think it's reasonable given the no-network constraint)
