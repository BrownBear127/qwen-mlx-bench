# Markdown-to-EPUB Batch Converter POC

A minimal proof-of-concept that batch-converts a directory of Markdown files
into a single EPUB ebook.

## Features

- **Dual-mode conversion**: Uses [pandoc](https://pandoc.org/) (CLI) when
  available, falls back to a pure-Python approach (`ebooklib` + `markdown`)
  otherwise.
- **Automatic discovery**: Finds all `.md` files in the `chapters/` directory.
- **Table of contents**: Generated automatically in both modes.

## Quick Start

### Option A — with pandoc (recommended, best output quality)

```bash
# Install pandoc first (https://pandoc.org/installing.html)
# Then:
uv run --no-project --with pandoc-cli python poc.py
```

### Option B — pure Python (no system dependencies)

```bash
uv run --no-project --with ebooklib,markdown python poc.py
```

## Project Structure

```
.
├── poc.py              # Main converter script
├── requirements.txt    # Python dependencies (for Option B)
├── README.md           # This file
├── chapters/           # Input Markdown files
│   ├── chapter1.md
│   ├── chapter2.md
│   └── chapter3.md
└── book.epub           # Output (generated by poc.py)
```

## How It Works

1. **Discovery** — Scans `chapters/` for all `*.md` files (sorted alphabetically).
2. **Conversion** —
   - **pandoc mode**: Calls `pandoc chapter1.md chapter2.md chapter3.md -o book.epub --toc`
   - **Python mode**: Parses each `.md` file with the `markdown` library, wraps the
     HTML in EPUB chapter containers via `ebooklib`, and writes the final `.epub`.
3. **Validation** — Checks that `book.epub` exists and is > 1 KB.

## Requirements

| Mode | Package | Install |
|------|---------|---------|
| pandoc | `pandoc-cli` | `uv pip install pandoc-cli` or install pandoc natively |
| Python fallback | `ebooklib`, `markdown` | Listed in `requirements.txt` |

## Customization

Edit the constants at the top of `poc.py`:

```python
CHAPTERS_DIR = "chapters"   # Input directory
OUTPUT_EPUB  = "book.epub"  # Output filename
BOOK_TITLE   = "My Markdown Book"
BOOK_AUTHOR  = "POC Author"
```
