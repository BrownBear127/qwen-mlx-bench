# Notes on Local LLM Serving Framework Tradeoffs

The local LLM serving landscape is dominated by a tradeoff between hardware breadth and ease of use. **llama.cpp** and **Ollama** are the most accessible options for consumer hardware, especially Apple Silicon, with llama.cpp offering the widest hardware support (x86, ARM, RISC-V, NVIDIA, AMD) and Ollama wrapping it in a simple Go-based CLI and API. **MLX/mlx-lm** is the best choice if you are exclusively on Apple Silicon, offering tight integration with the Metal GPU stack and a NumPy-like Python API, but it lacks cross-platform GPU support. For production-grade throughput on NVIDIA GPUs, **vLLM** and **SGLang** lead with advanced features like PagedAttention, continuous batching, and speculative decoding, though both are primarily GPU-focused (vLLM has experimental Apple Silicon support). **Hugging Face TGI** is now in maintenance mode but remains relevant for Rust-based deployments with strong quantization support; however, it requires dedicated GPU hardware and does not support Apple Silicon. All frameworks now support tool calling to varying degrees, typically through OpenAI-compatible API endpoints, making them viable for agentic workflows.
