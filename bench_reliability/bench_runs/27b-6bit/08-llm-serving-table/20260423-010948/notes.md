# Notes on Local LLM Serving Frameworks

The local LLM serving landscape is dominated by two distinct design philosophies: lightweight, hardware-agnostic runtimes like **llama.cpp** (and its wrappers Ollama and llama-cpp-python) that prioritize broad device support and aggressive quantization via the GGUF format, versus high-throughput GPU-centric engines like **vLLM** and **SGLang** that leverage PagedAttention, continuous batching, and tensor parallelism for production-scale serving. **MLX** occupies a unique niche as Apple's native framework — it delivers excellent performance on Apple Silicon with a clean NumPy-like API, but is limited to macOS/iOS devices and lacks built-in tool-calling support. For developers targeting Apple Silicon specifically, llama.cpp and MLX are the strongest choices; for multi-GPU server deployments, vLLM and SGLang offer superior throughput and richer quantization options. Ollama sits in the middle as the most user-friendly option, abstracting llama.cpp's complexity behind a simple CLI and REST API while still supporting tool calling and all GGUF quantization formats.
