The local LLM serving landscape is dominated by llama.cpp and its derivatives, which provide the most mature and widely adopted on-device inference stack. llama.cpp stands out for its broad hardware support (Apple Silicon, NVIDIA, AMD, RISC-V) and extensive quantization options (down to 1.5-bit), making it the de facto engine for most other frameworks. Ollama wraps llama.cpp in a polished Go-based CLI and API, adding convenience features like one-command model pulling and tool calling with streaming, but at the cost of less low-level control. Apple's MLX is purpose-built exclusively for Apple Silicon, offering excellent performance and a clean NumPy-like API, but it lacks cross-platform portability and does not include built-in tool calling at the framework level. llamafile takes a unique approach by bundling llama.cpp into a single self-contained executable via Cosmopolitan Libc, prioritizing distribution simplicity over feature breadth. Finally, llama-cpp-python provides the most developer-friendly Python interface to llama.cpp, with first-class function calling support and OpenAI API compatibility, making it ideal for integration into Python-based AI applications.
